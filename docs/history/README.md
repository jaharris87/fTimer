# Historical Planning Records

This directory holds issue-era design decisions, validation plans, and
implementation-planning notes that are useful for archaeology but are not the
current documentation surface.

For current behavior, use:

- [`README.md`](../../README.md) for user-facing setup, examples, and
  limitations
- [`docs/semantics.md`](../semantics.md) for runtime contracts
- [`docs/design.md`](../design.md) for current architecture and CI reality
- [`docs/installed-api.md`](../installed-api.md) for stable source-level API
- [`docs/openmp-timing-modes.md`](../openmp-timing-modes.md) for OpenMP and
  MPI+OpenMP mode selection and migration guidance

Historical records here may mention superseded issue numbers, staged follow-up
plans, or implementation expectations that were later narrowed or completed.
When these records disagree with current code, tests, or the regular
documentation surface, the source-of-truth order in `AGENTS.md` applies.
